# Галерея художника — В память

Сайт-галерея, посвящённый памяти художника **Владимира Бакуменко**. Одностраничное веб-приложение на русском языке: биография, разделы работ (акварель и масло), контакты и страница памяти.

---

## О проекте

Проект создан как мемориальный сайт: показать творческое наследие художника, дать возможность просматривать работы по категориям и связаться с близкими по вопросам картин и сотрудничества.

**Основные разделы:**

| Раздел | Описание |
|--------|----------|
| **Главная** | Приветственный экран с фото автора, карусель работ и быстрые ссылки на разделы галереи |
| **Галерея** | Два подраздела — **Акварель** (36 работ) и **Масло** (6 работ). Сетка карточек, ленивая подгрузка, лайтбокс для просмотра в полном размере |
| **О художнике** | Краткая информация об авторе и творческом пути |
| **В память** | Текст и цитаты в память о художнике |
| **Контакты** | Email, телефон и Telegram для связи |

Интерфейс на тёмном фоне, с аккуратными переходами и декоративными элементами (блоки фона, боке на странице «О художнике»).

---

## Технологии

- **React 19** + **TypeScript**
- **Vite 7** — сборка и dev-сервер
- **React Router 7** — маршрутизация (`/`, `/gallery/akvarel`, `/gallery/maslo`, `/about`, `/memory`, `/contacts`)
- **SCSS (модули)** — стили страниц и компонентов
- **Tailwind CSS 4** — утилитарные классы
- **Embla Carousel** — карусель на главной

Шрифты: Cormorant Garamond, Source Sans 3 (Google Fonts).

## Запуск

```bash
# Установка зависимостей
npm install

# Режим разработки (обычно http://localhost:5173)
npm run dev

# Сборка для продакшена
npm run build

# Просмотр собранной версии
npm run preview
```

## Замечания по репозиторию

В репозитории хранятся десятки изображений (40+ фото) в `public/assets/`. Их объём может приводить к большим коммитам и к ошибкам при `git push` (HTTP 400, обрыв соединения). Рекомендации:

- Увеличить буфер: `git config http.postBuffer 524288000`
- Для дальнейшей работы — сжимать изображения перед коммитом или вынести их на CDN/хостинг и подставлять URL в коде.
